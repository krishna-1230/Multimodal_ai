HTML to PDF API Documentation

Introduction

This API allows you to convert HTML content to PDF and optionally upload it to Google Drive. The service provides two main endpoints: one for direct PDF conversion and another for conversion with Google Drive upload.

Authentication

No authentication is required for the basic conversion endpoint. For the Google Drive upload endpoint, the server uses pre-configured Google API credentials.

Endpoints

POST /convert

Converts HTML content to PDF and returns the PDF file directly in the response.

Request Body

Parameter Type Description Required
html string The HTML content to convert to PDF Yes

Example Request


POST /convert
Content-Type: application/json

{
    "html": "<h1>Hello World</h1><p>This is a sample HTML content.</p>"
}

Response

Returns the PDF file with Content-Type: application/pdf

POST /api/convert-local

Converts HTML content to PDF and saves it locally on the server, returning a direct download URL. Ideal for n8n integration.

Request Body

Parameter Type Description Required
html string The HTML content to convert to PDF Yes
fileName string Name for the PDF file (defaults to timestamp-based name) No

Example Request


POST /api/convert-local
Content-Type: application/json

{
    "html": "<h1>Hello World</h1><p>This is a sample HTML content.</p>",
    "fileName": "my-document.pdf"
}

Response

Success Response (200 OK):


{
    "success": true,
    "file": {
        "name": "abc123-my-document.pdf",
        "path": "/path/to/public/uploads/abc123-my-document.pdf",
        "downloadUrl": "http://localhost:3000/uploads/abc123-my-document.pdf"
    }
}

Error Response (400 Bad Request):


{
    "success": false,
    "message": "HTML content is required"
}

Error Response (500 Internal Server Error):


{
    "success": false,
    "message": "Error converting and saving PDF",
    "error": "Error details here"
}

POST /api/convert-and-upload

Converts HTML content to PDF and uploads it to Google Drive, returning links to access the file.

Request Body

Parameter Type Description Required
html string The HTML content to convert to PDF Yes
fileName string Name for the PDF file in Google Drive (defaults to timestamp-based name) No

Example Request


POST /api/convert-and-upload
Content-Type: application/json

{
    "html": "<h1>Hello World</h1><p>This is a sample HTML content.</p>",
    "fileName": "my-document.pdf"
}

Response

Success Response (200 OK):


{
    "success": true,
    "file": {
        "id": "1aBcDeFgHiJkLmNoPqRsTuVwXyZ",
        "name": "my-document.pdf",
        "viewLink": "https://drive.google.com/file/d/1aBcDeFgHiJkLmNoPqRsTuVwXyZ/view",
        "downloadLink": "https://drive.google.com/uc?id=1aBcDeFgHiJkLmNoPqRsTuVwXyZ&export=download"
    }
}

Error Response (400 Bad Request):


{
    "success": false,
    "message": "HTML content is required"
}

Error Response (500 Internal Server Error):


{
    "success": false,
    "message": "Error converting and uploading PDF",
    "error": "Error details here"
}

Setup Requirements

To use the Google Drive upload functionality, you need to:

  1. Create a Google Cloud project and enable the Google Drive API
  2. Create OAuth 2.0 credentials (client ID and client secret)
  3. Obtain a refresh token for your Google account
  4. Configure these credentials in the config.env file

For more information on setting up Google Drive API, refer to the Google Drive API documentation.
